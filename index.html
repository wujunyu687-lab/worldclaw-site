PROJECT CONTROL PLANE

WorldClaw brings training, evaluation, paper search, and multi-platform bot routing into one project workspace.

It is not a single training repo, and it is not just a chat shell. WorldClaw is a project-level control plane: terminal, WeChat, and Feishu share one execution layer, while TrainBot, EvalBot, PaperSearch, and Report workflows share the same workspace, runtime, and memory layer. This public site only exposes the project architecture and public-facing overview. The working implementation repository can remain private.

UNIFIED ACCESS

Three access surfaces, one execution layer

Entry bots receive user messages. The actual work is delegated to internal web bots. This keeps platform adaptation separate from training, evaluation, search, and reporting logic.

Terminal

`bin/worldclaw` is the direct local interface for development, bot switching, status checks, and session continuation. It prefers existing web instances before starting local ones. 

/home/hywu/wjy/worldclaw/bin/worldclaw --bot train-bot "List variants for demo-mock"

WeChat

`metabot` and `metabot-2` are WeChat entry bots. They do not contain all business logic themselves. They route requests to train, eval, papersearch, and report bots behind the bridge. 

Call train-bot and run a smoke dry-run for the demo-mock smoke variant

Feishu

`worldclaw-feishu` is the Feishu long-connection entry bot. Like the WeChat layer, it is a platform bridge rather than a separate business implementation. 

Call eval-bot, run a WorldArena precheck, and report missing environments and weights

CAPABILITY LAYERS

This is not a pile of scripts. It is a layered control plane.

WorldClaw separates training control, evaluation control, paper search, and reporting into dedicated modules. Each module keeps its own working directory, docs, and execution constraints, while still being routed through the same orchestrator.

trainbot/ Control Plane

TrainBot

A general training control layer built around targets, variants, experiments, ledgers, and lineage, without binding the system to a single training repository.

  • variant registry
  • alias & inheritance
  • smoke-variant
  • resolved-config
  • compare / latest-summary
evalbot/ Benchmark Entry

EvalBot

A general evaluation control layer for benchmark entry points, input requirements, environment validation, weight readiness, execution, and result reporting. WorldArena is the first major entry currently wired in.

  • benchmark registry
  • precheck
  • environment validation
  • weight readiness
  • result location reporting
Paper_searchbot/ Research Ops

PaperSearch

Responsible for topic search, conference proceedings crawling, title-and-abstract collection, CSV normalization, and merged research outputs.

  • topic search
  • proceedings crawl
  • CSV normalize
  • merge outputs
  • project-ready outputs
metabot/ Bridge Layer

MetaBot Bridge

Handles the IM bridge, local API, web UI, message dispatch, and memory service. Business logic is not hard-coded into the platform integration layer.

  • wechat / feishu
  • terminal API
  • memory service
  • bot registry
  • shared runtime

BOT TOPOLOGY

Access stays separate from execution, so the system scales cleanly.

Entry bots solve platform access and session intake. Internal web bots handle training, evaluation, paper search, and reporting. The result is more stable than pushing every capability into one chat-facing bot, and much easier to maintain in GitHub.

WORKFLOW

WorldClaw is built to connect the full project loop.

The private implementation may live elsewhere, but the public site still explains the real operational loop instead of collapsing everything into vague marketing copy.

01

Define the target

Describe the training project with TrainBot target, variant, and experiment contracts instead of scattering configuration across prompts and scripts.

02

Run training

Execute stage graphs, smoke variants, and full experiments, then persist summaries, bundles, resolved configs, and lineage.

03

Run evaluation

EvalBot starts from benchmark selection and prechecks, explains environments and weights, then reports both results and saved output locations.

04

Search and report

PaperSearch collects related work, ReportBot produces status reports and archives, and MetaMemory keeps shared context persistent.
